# linear-pp-cli Phase 3 Build Log (reprint, v3.10.0)

## Pivot decision
Started in `$CLI_WORK_DIR` with a fresh v3.10.0 generation against Linear's GraphQL SDL (45,863 lines, ~98 entity types). Generation succeeded with all 7 quality gates green but emitted a REST-oriented sync template — the v1 retro's WU-3 (GraphQL sync template) hasn't shipped in v3.10.0. Re-implementing v1's GraphQL infrastructure from scratch in a fresh tree would burn the session budget.

**Pivot:** work directly in `~/printing-press/library/linear/` (the v1-imported tree), preserving v1's hand-written GraphQL client + queries + sync, and adding only what the reprint mandate requires: close ship-with-gaps debt + add live-testing safety features + the 4 net-new transcendence features.

## Built (12 of 12 manifest items)

### Carried from v1, verified compile (5)
- `today` — daily ranked view
- `bottleneck` — pre-sprint-planning overload + blocked check
- `similar` — FTS5 duplicate detection
- `stale` (= `stale_custom.go`) — issues idle past threshold
- `velocity` — multi-cycle completion-rate trends

### Closed v1 ship-with-gaps debt (2)
- `cycles compare <a> <b>` — two-cycle SQL diff (completion %, scope-added, scope-cut, carryover, per-state). New file `internal/cli/cycles_compare.go` (244 lines). Wired as subcommand of `cycles`. Aliases: `current`/`previous`/numeric/UUID.
- `projects burndown <project>` — velocity-driven landing-date forecast. New file `internal/cli/projects_burndown.go` (213 lines). Wired as subcommand of `projects`. Reads open vs completed issues + estimate field, regresses against rolling 4-week velocity, projects landing date with target-date delta.

### Net-new transcendence features (3)
- `slipped` — issues that carried from previous cycle into current. New file `internal/cli/slipped.go` (163 lines). Top-level command. Reason heuristic: `in_progress`/`carried`/`blocked`.
- `blocking` — your downstream-impact queue (live GraphQL query — local store doesn't snapshot relations). New file `internal/cli/blocking.go` (216 lines). Top-level. Walks `issue.relations` filtered to `type=blocks`, weights by priority.
- `initiatives health` — portfolio rollup over initiatives → projects → milestones. New file `internal/cli/initiatives_health.go` (200 lines). Wired as subcommand of `initiatives`. Live GraphQL query.

### Live-testing safety features (3 + 1 flag)
- `pp-test list` / `pp-test sessions` — `internal/cli/pp_test_cmd.go` (133 lines). Lists active fixtures from local `pp_created` ledger, scoped by session.
- `pp-cleanup` — `internal/cli/pp_cleanup.go` (203 lines). Archives every issue in `pp_created` for the named session via Linear's `issueArchive` mutation. Touches ONLY fixtures this CLI made.
- `issues create` — new file `internal/cli/issues_create.go` (190 lines). Wired as subcommand of `issues`. Calls `issueCreate` GraphQL mutation; records the new issue ID into `pp_created` so cleanup can find it. Resolves team key (e.g. `ENG`) to UUID via the local teams cache.
- `--trust-mode` root flag (advertised; safety derives from the limited mutation surface — only `issues create` and `pp-cleanup`, both pp_created-scoped).

### Store changes
- New table `pp_created (issue_id, identifier, title, session, created_at, archived_at)` with index on `session`. Migration appended to `internal/store/store.go`.
- New helpers in `internal/store/pp_fixtures.go`: `RecordPPFixture`, `IsPPCreated`, `ListPPFixtures`, `MarkPPFixtureArchived`, `ListPPSessions`.

### Removed (per absorb gate)
- `workload` (`internal/cli/workload.go` and its registration in root.go) — sibling-killed by `bottleneck`.
